# PretextChat 路线图

## Phase 1：验证产品成立

### 阶段目标

验证用户是否愿意用 PretextChat 替代浏览器中的 AI 标签页，把它作为 **AI 主入口 + 多实例管理器**。

### 核心判断标准

- 用户是否愿意从这里进入 AI
- 用户是否会持续打开多个 AI 实例
- 用户是否会依赖“恢复上次状态”
- 用户是否认为这比普通浏览器更适合管理 AI 任务

### P0

- 预置 AI 应用目录
- 点击 AI 应用即创建实例
- 新建 / 切换 / 关闭 / 重命名实例
- 基础标签页系统
- 最近一次会话恢复
- 登录态保持

### P1

- 最近使用的 AI
- 快速重开最近实例
- 启动方式选择：进入首页 / 恢复上次会话
- 实例默认命名策略
- 应用图标与名称统一视觉规范

### P2

- 最近关闭实例恢复
- 应用收藏 / 常用应用置顶

## Phase 2：进入工作台阶段

### 阶段目标

把产品从“AI 标签管理器”推进到“AI 并行工作台”，开始体现 Pretext 的交互性能价值和重度用户效率价值。

### 核心判断标准

- 用户是否开始在 PretextChat 内持续停留工作
- 用户是否开始并行使用多个 AI
- 用户是否明显感知到切换、输入、长内容场景更顺

### P0

- 工作区单栏 / 双栏
- 双栏模式下实例分配
- 统一 Prompt Bar：发送到当前实例
- 长会话场景稳定性优化

### P1

- 标签拖拽排序
- 固定标签
- 最近关闭实例恢复
- 快捷键体系
- Command Palette
- 标签处理中提示 / 未读提示
- 应用收藏 / 置顶

### P2

- Prompt 草稿短暂保留
- 分屏比例拖拽
- 更丰富的启动页信息

## Phase 3：形成差异化壁垒

### 阶段目标

让 PretextChat 从“好用的 AI 工作台”变成“有明显壁垒的 AI 工作操作系统”。

### 核心判断标准

- 用户是否开始依赖 PretextChat 进行复杂 AI 工作编排
- 用户是否愿意长期沉淀自己的 AI 使用结构
- 产品是否与普通 AI 客户端明显拉开差距

### P0

- 一问多发
- 多实例结果对比视图
- 多工作区

### P1

- 实例分组
- 独立窗口 / 多窗口
- 自定义 AI 应用入口
- 团队级应用预设
- Prompt 模板
- 实例模板
- 工作区模板

### P2

- 本地导入 / 分享应用配置
- 跨设备同步
- 更深入的 Pretext 文本性能缓存策略
