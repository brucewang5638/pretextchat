# 官网静态站部署说明

## 目录位置

- 官网源码：`website/`
- 构建命令：`npm run site:build`
- 构建产物：`website/dist/`

这个目录与 Electron 主应用解耦，适合单独托管到静态站平台。

## 平台建议

### 1. 首选：腾讯云 EdgeOne Pages

适合你现在这个项目的原因：

- 体验上最像 Vercel：支持 Git 仓库自动部署、预览环境、自定义域名。
- 腾讯官方文档明确写到可以连接 Git 仓库部署，并在项目设置中配置自定义域名。
- Pages FAQ 明确写到：添加自定义域名后，平台会自动提供 SSL 证书。
- 官方 FAQ 还直接提到，在中国地区相较 Vercel、Cloudflare Pages 有明显优势。

适合场景：

- 想保留现代前端部署体验
- 希望后续继续通过 Git 自动发布官网
- 需要比海外平台更好的大陆访问体验

### 2. 更稳的大陆生产方案：腾讯云 COS 静态网站 + CDN

适合你在下面这些情况下使用：

- “大陆访问稳定”比“部署体验像 Vercel”更重要
- 可以接受多一点云资源配置
- 希望把 CDN、证书、域名、缓存规则拆开精细控制

腾讯云文档明确支持：

- COS 托管静态网站
- 绑定自定义域名
- 配置 HTTPS
- 搭配 CDN 加速

这个方案更偏传统，但对正式官网会更稳。

### 3. 阿里云 OSS 静态网站 + CDN

如果你后续域名、备案、CDN 都打算放在阿里云，这也是非常稳的一条线。

阿里云官方文档明确支持：

- OSS 静态网站托管
- 自定义域名访问
- 中国内地 Bucket 场景下需要 ICP 备案

如果你团队更熟阿里云，这个方案完全可行。

### 4. 不建议作为主站：GitHub Pages

GitHub Pages 官方确实支持：

- 自定义域名
- HTTPS

但如果“大陆可访问”是硬要求，我不建议把它作为主站。它更适合做备用文档站、镜像页或海外开发者说明页。

## 我的建议

如果你要一个最平衡的选择，我建议：

1. 先用 `website/` 这套静态站源码。
2. 部署优先选 `腾讯云 EdgeOne Pages`。
3. 域名尽量放在腾讯云或阿里云统一管理。
4. 如果你准备长期面向中国大陆正式运营，就尽早做 ICP 备案。

如果你更看重“生产稳定性第一”，那就直接走：

`website/dist` -> `COS/OSS` -> `CDN` -> `自定义域名`

## 备案提醒

这是这类站点最容易忽略的一点：

- 腾讯云 ICP 文档明确说明：网站/域名接入中国内地服务时，需要完成备案流程。
- 腾讯云 EdgeOne 文档明确说明：当加速区域为“中国大陆可用区”或“全球可用区”时，域名必须完成 ICP 备案。
- 阿里云 OSS 文档也明确说明：如果 Bucket 位于中国内地并绑定自定义域名，需要 ICP 备案。

所以可以简单记成一句话：

**只要你想认真做大陆访问，不要把备案当可选项。**

## 上线建议

### 快速上线版

- 保留 `website/`
- `npm run site:build`
- 接入 EdgeOne Pages
- 绑定自定义域名
- 先上线，再补充监测与埋点

### 稳定生产版

- 保留 `website/`
- `npm run site:build`
- 上传 `website/dist/` 到 COS 或 OSS
- 开 CDN
- 绑定域名和 HTTPS
- 做 ICP 备案
- 最后接入搜索站长平台和基础统计
